# Strict Mathematical Correctness Audit Report

Audit date: 2026-06-13. Source: local course PDF (not included in the public repository).

This audit compares the generated study system against the PDF text layer and the generated source inventory, then records targeted PDF-page checks. It does **not** certify full mathematical reliability; formulas with PDF text-layer corruption still need visual checking against the PDF.

## Executive Summary

- Sampled/check rows recorded: 363.
- Correct: 303.
- Minor issues: 46.
- Serious issues: 10.
- Unverifiable/deficits: 4.
- Uncertainty-marked source entries checked: 45.

## Main Findings

1. **Serious, fixed:** Appendix A was missing from the generated databases because labels like `A.1.1` were not parsed.
2. **Serious, fixed:** `scripts/source-inventory.md` was missing; only root `source-inventory.json` existed.
3. **Serious, fixed:** eight high-yield overlays were attached to wrong IDs, causing incorrect labels/formal summaries in definitions, theorems, proof training, questions, flashcards, and app data.
4. **Minor, fixed:** unnamed definitions no longer fall back to broad section titles when a term can be inferred from the definition text.
5. **Minor, partly fixed:** common extraction artifacts for roots, arrows, empty set, set operations, and appendix labels were normalized, but some formulas remain OCR-derived and must be visually verified.
6. **Minor, not fully fixable automatically:** many proof-database entries are proof scaffolds plus OCR proof transcriptions, not polished proof rewrites. High-yield proof trainers are more useful, but the proof database should not be treated as exam-ready without PDF comparison.

## Corrections Applied

- Updated `scripts/build_study_system.py` to parse Appendix A labels.
- Regenerated the package; inventory now has 404 detected numbered items.
- Added `scripts/source-inventory.md`.
- Corrected high-yield overlays: `3.2.3`, `4.2.1`, `4.3.2`, `5.2.4`, `5.3.5`, `6.4.2`, `6.6.5`, `7.2.9`.
- Replaced broad fallback labels for unnamed definitions with inferred terms or `unnamed definition in <section>`.
- Regenerated flashcards; CSV now validates with 401 cards.
